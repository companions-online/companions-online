# user-guide — Docusaurus site

Player + AI-companion documentation. Builds to `../docs/` for
GitHub Pages (org-level URL).

## Source dir vs URL

- **Source**: `user-guide/`
- **URL base**: `/guide/` (set via `routeBasePath: 'guide'` in
  `docusaurus.config.ts`; `docsRouteBasePath: '/guide'` in the
  search plugin). The two intentionally differ — don't try to
  unify.

## Sidebar shape

Autogenerated from numeric-prefixed dirs/files under `docs/`:

```
  01-controls/           (Controls)
    01-desktop.md        (filled)
    02-mobile.md         (placeholder; tap-driven, doc when ready)
  02-survival-basics.md
  03-crafting.md
  04-building.md
  05-server-commands.md
02-self-host.md
03-ai-companions/        (AI companions)
  01-concept.md … 07-mmo-bench.md
04-contribute/           (Contribute)
  01-local-setup.md
  02-community.md
```

Each subsection has `_category_.json` with an explicit `slug` to
dodge Docusaurus's default `/category/<dir>` URL prefix — the
indices live at `/guide/player-guide`, `/guide/ai-companions`,
etc. Don't drop the `slug` field; broken links return.

## Build

`npm run build:guide` from repo root:

1. `npm run build:client-gl` — bundles the WebGL client to
   `client-webgl/dist/main.js`.
2. `npm --prefix user-guide install`.
3. Docusaurus build runs `prebuild` → `scripts/copy-assets.mjs` →
   `docusaurus build --out-dir ../docs`.

`copy-assets.mjs` is the cross-project integration seam. It pulls:
- `client-webgl/assets/*` → `user-guide/static/assets/`
- `client-webgl/dist/main.js[.map]` → `user-guide/static/game/`
- `harness/config/prompt.md` → `user-guide/static/prompt.md`

All three are gitignored under `static/`. Adding a new
cross-project artifact = extend that script, not commit a
duplicate.

## Recipe icons

`user-guide/static/img/recipes/<kebab-name>.png`. **One-time
extraction** (Python+PIL, ad-hoc), not part of the build. Five are
resource stand-ins: `iron-sword/spear/chestplate.png` are
`iron.png`; `wooden-floor.png` is `wood.png`; `stone-floor.png` is
`rock.png`. When the WebGL client ships real icons for those,
overwrite the files in `static/img/recipes/`.

## Live prompt link

`docs/03-ai-companions/04-prompting.md` links
`pathname:///prompt.md`. The file is the build-time copy of
`harness/config/prompt.md`. To change what ships on the published
guide, edit the harness prompt and rebuild — never edit
`static/prompt.md` directly (it's regenerated and gitignored).

## Editorial conventions

- §1–§3 (player guide, self-host) friendly tone; §4 AI companions
  + §5 contribute lean reference/neutral.
- "Companions" = motivational/marketing copy; "AI player" /
  "LLM player" / "MCP client" in technical sections.
- Pixel-art icons use `image-rendering: pixelated` via
  `.recipe-icon` in `src/css/custom.css`.

## Don't double-document

The docs are the source of truth for player- and AI-facing
content. Don't re-encode controls, recipes, MCP tool list, etc.
into memory — they go stale and silently mislead.
